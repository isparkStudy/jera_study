java.lang : 자바의 언어와 관련된 기본 클래스 제공
java.util : 자료구조와 관련돈 클래스 제공
java.io : 입출력 관련 클래스 제공
java.net : 네트워크와 관련된 클래스 제공

동등 비교 기능이 필요하다면 equals 를 재정의해서 써야한다. 

DTO 설계 시 관례.
1. 필드는 private로 정의한다.
2. 필드 값은 게터와 세터를 만들어서 작성한다.
 * 레코드로 선언 시 상기 관례가 자동으로 추가된다.

롬복
: 레코드와 동일한 목적으로 사용되는 라이브러리.
: 표준 라이브러리가 아니어서 별도로 다운받아야함.

레코드와의 차이점
- 게터와 세터를 DTD 설계 관례에 맞게 설계해줌
- 필드 선언 시 final이 아님

인코딩 : 문자열 -> 바이트 변환
디코딩 : 바이트 -> 문자열 변환

String Builder 클래스
: 문자열을 완성해가는 방식

Calender 클래스
: 추상클래스로 생성자를 이용하여 객체를 생성 할 수 없음. 정적 메소드를 이용해야함.
